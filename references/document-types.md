# Common Document Types (常用文档类型)

## 公文报告 (Official Report)

**结构**：
```
标题
摘要
一、背景
二、主要内容
三、结论与建议
```

**排版要点**：
- 标题居中，三号黑体
- 正文小四宋体，首行缩进
- 一级编号：一、二、三
- 包含表格、数据时使用 Word 表格

## 个人简历 (Resume)

**结构**：
```
姓名 + 联系方式
教育背景
工作经历
专业技能
```

**排版要点**：
- 简洁紧凑，控制在一页内
- 标题用加粗区分，不用大号字体
- 时间线右对齐

## 会议纪要 (Meeting Minutes)

**结构**：
```
会议主题 + 基本信息（时间/地点/参会人）
议题列表
讨论要点
决议事项
后续行动（负责人 + 截止日期）
```

**排版要点**：
- 基本信息（时间地点等）在标题下方
- 讨论内容按议题分组
- 行动项用表格列出

## 工作总结 (Work Summary)

**结构**：
```
标题（含时间范围）
一、本期工作回顾
二、重点项目进展
三、问题与挑战
四、下期工作计划
```

**排版要点**：
- 数据用表格呈现
- 进展用列表展示
- 问题部分简明扼要

## 方案建议书 (Proposal)

**结构**：
```
标题
一、背景与目标
二、方案详述
三、实施计划
四、资源需求
五、预期收益
六、风险评估
```

**排版要点**：
- 方案详述是主体，篇幅最长
- 实施计划可用表格（阶段/时间/交付物）
- 风险评估建议用表格（风险/等级/应对措施）
